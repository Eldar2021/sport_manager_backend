# Managers REST API — Контракт бэкенда

Список менеджеров, управляемых владельцем. Мобильный клиент (`packages/managers`) использует эти эндпоинты.

**Base URL:** `<BASE_URL>`
**Content-Type:** `application/json; charset=utf-8`
**Auth:** Все эндпоинты требуют `Authorization: Bearer <accessToken>`, **role = `OWNER`**. Не-OWNER должен получить `403`.

> Invite-код (для отображения) принадлежит отдельному эндпоинту: `POST /api/v1/auth/invite-code` (см. [auth-api.md](auth-api.md#6-post-apiv1authinvite-code)). Экран менеджеров переиспользует этот существующий эндпоинт.
>
> Эндпоинта **ротации / обновления** invite-кода пока нет — соответственно, в клиенте нет кнопки "обновить". Если добавите, рекомендуем `POST /api/v1/auth/invite-code/refresh`.

## 1. GET `/api/v1/managers`

Список менеджеров, привязанных к владельцу.

### Запрос

Query-параметры отсутствуют. Тело отсутствует.

### 200 OK — тело ответа

```json
[
  {
    "id": "user-101",
    "name": "Айбек Асанов",
    "username": "aibek",
    "lastSeenAt": "2026-04-30T08:15:00Z"
  },
  {
    "id": "user-102",
    "name": "Нурлан Беков",
    "username": "nurlan",
    "lastSeenAt": "2026-04-28T19:42:00Z"
  },
  {
    "id": "user-103",
    "name": "Данияр Токтогул",
    "username": "daniyar",
    "lastSeenAt": null
  }
]
```

### Ошибки

| HTTP | `code`      | Причина                  |
| ---- | ----------- | ------------------------ |
| 401  | —           | Нет токена / истёк       |
| 403  | `FORBIDDEN` | Пользователь не OWNER    |

## 2. DELETE `/api/v1/managers/{id}`

Удаляет указанного менеджера из команды владельца. Поведение бэкенда при наличии у менеджера активных сессий — на усмотрение бэкенда (можно закрыть или вернуть `409`).

### Запрос

Тело отсутствует.

### 200 OK / 204 No Content

Тело не ожидается.

### Ошибки

| HTTP | `code`               | Причина                                                  |
| ---- | -------------------- | -------------------------------------------------------- |
| 401  | —                    | Нет токена / истёк                                      |
| 403  | `FORBIDDEN`          | Пользователь не OWNER или менеджер другого владельца   |
| 404  | `MANAGER_NOT_FOUND`  | Менеджер с таким id не существует                        |
| 409  | `HAS_ACTIVE_SESSION` | У менеджера есть активная сессия (опционально)          |

## Models — Справочник полей

### `ManagerModel`

```json
{
  "id": "user-101",
  "name": "Айбек Асанов",
  "username": "aibek",
  "lastSeenAt": "2026-04-30T08:15:00Z"
}
```

| Поле         | Тип              | Обязательное | Примечания                                                  |
| ------------ | ---------------- | ------------ | ----------------------------------------------------------- |
| `id`         | string           | да           | Генерируется сервером, стабильный                           |
| `name`       | string           | да           | Отображаемое имя                                            |
| `username`   | string           | да           | Без `@`; в UI отображается как `@username`                  |
| `lastSeenAt` | ISO-8601 \| null | нет          | Последний раз активен; null → "ещё не появлялся"           |

## Формат ответа с ошибкой

Тот же конверт, что в `auth-api.md`:

```json
{
  "code": "MANAGER_NOT_FOUND",
  "message": {
    "en": "Manager not found",
    "ru": "Менеджер не найден",
    "ky": "Менеджер табылган жок"
  },
  "details": null
}
```

Карта `code` бэкенда → enum `ManagerErrorCode` на клиенте:

| `code` бэкенда       | `ManagerErrorCode` клиента | Типичный HTTP |
| -------------------- | -------------------------- | ------------- |
| `MANAGER_NOT_FOUND`  | `notFound`                 | 404           |
| `FORBIDDEN`          | `forbidden`                | 403           |
| `HAS_ACTIVE_SESSION` | `hasActiveSession`         | 409           |
| (всё остальное)      | `unknown`                  | 4xx/5xx       |
